Why UNIX?
This is a very obvious question for the developers who are used to work on Windows. So when I started working on Unix , the first thing I did was to get an answer of "Why Unix"?

 UNIX is "multi-user".
Unlike MS-DOS, OS/2, and Windows NT, UNIX is a multi-user operating system. This means that multiple users can log in to a computer at one time, and use the resources of that computer. Resources can be a machine, CPU, software applications, as well as a hard disk drives, printers, modems, etc.

 I have tried out these very basic  Unix commands and think they would be of help for the beginners in this field.

 Useful ls commands -
  • Open Last Edited File Using ls -t
  • Display One File Per Line Using ls -1
  • Display All Information About Files/Directories Using ls -l
  • Display File Size in Human Readable Format Using ls -lh
  • Display Directory Information Using ls -ld
  • Order Files Based on Last Modified Time Using ls -lt
  • Order Files Based on Last Modified Time (In Reverse Order) Using ls -ltr
  • Display Hidden Files Using ls -a (or) ls -A
  • Display Files Recursively Using ls -R
  
UNIX Control-Key Commands
  • Ctrl-h : Erase or backspace over character; note the CTRL-key and h-key must be simultaneously pressed.
  • Ctrl-c : Interrupt or break character; stops printing and returns to UNIX. 
  • Ctrl-u : Kill character .
  • Ctrl-w : Word erase character .

 Redirecting input & output:
command1 > file1 executes command1, placing the output in file1. This will clobber any existing data in file1.
cat >> file1.txt
To append output to the end of the file, use the >> operator
cat < file1.txt > file2.txt 
combines the two capabilities: command1 reads from file1.txt and writes to file2.txt

Redirect to multiple outputs
ls -lrt | tee xyz
The standard command tee can redirect output from a command to several destinations.This directs the file list output to both standard output and the file xyz.
